   1. FE1 Моніторинг безпечних умов експонатів у реальному часі: ArtGuard
       забезпечує безперервний моніторинг ключових показників (температура,
       вологість, вібрація) для кожного експоната. Дані надходять у режимі
       реального часу, що дозволяє оперативно відстежувати стан об'єктів..



       Виконано: На платформі Wokwi була реалізована симуляція функціонування
IoT,   що   надає   можливість    слідкувати   за   експонатами музею. Завдяки
використанню серверного рішення на основі Ngrok, стало можливим безпосереднє
надсилання HTTP-запитів до бази даних, що забезпечує передачу даних у
реальному часі — дані оновлюються через визначений проміжок часу. Крім того,
був розроблений GET-запит для реалізації отримання статистики сенсорів.

   2. FE2 Інтерактивна система сповіщень: У разі перевищення допустимих
       меж параметрів система автоматично надсилає сповіщення на мобільні
       пристрої та веб-інтерфейс. Це дозволяє співробітникам музею швидко
       реагувати на потенційні загрози.


       Виконано: API для відстеження алертів(сповіщень). Працівники музею
можуть отримувати сповіщення, а у майбутньому через мобільний застосунок
побачити потрібний їм експонат та перевірити його. Дані записуються у базу
даних, користувачі отримують підтвердження про небезпеку.
  3. FE3 Історія даних та аналітика: ArtGuard зберігає історію всіх
     кліматичних    даних   для   кожного   об’єкта,   що   дає   змогу   музеям
     відслідковувати зміни та аналізувати тренди в умовах зберігання,
     допомагаючи ухвалювати більш обґрунтовані рішення щодо управління
     експонатами.


     Виконано: За допомогою створеною бізнес логіки проєкту, можливо
отримувати детальну статистику про ризики та небезпеки, наразі через API запити.


  4. FE4    Ролі   користувачів:   Визначення    ролей    з    відповідними
     функціональними можливостями для керування та заведення акаунтів.

     Виконано: Реалізовано різний функціонал для різних ролей робітників
музею, на даний момент це звичайний користувач та адміністратор, який має право
керувати іншими користувачами та змінювати особисто важливі дані.
  5. FE5 Мобільний та веб-інтерфейс: ArtGuard надає зручний мобільний
     додаток для отримання сповіщень у режимі реального часу, а також
     веб-інтерфейс для управління всіма параметрами та перегляду аналітики.

     Нереалізовано: На даний момент проєкт не має мобільного застосунку та
веб-інтерфейсу.
  6. FE6 Персоналізовані налаштування для зон зберігання: Система
     дозволяє створювати персоналізовані налаштування для різних зон або
     категорій об’єктів, враховуючи індивідуальні вимоги для зберігання
     конкретних типів мистецьких об’єктів. ArtGuard надає музеям/установам
     автоматизовану систему, що забезпечує моніторинг та зберігання даних про
     кліматичні умови. Дані з IoT-сенсорів передаються на серверну частину, де
     обробляються й зберігаються для подальшого аналізу. Якщо показники
     кліматичних умов виходять за межі допустимих значень, система
     автоматично генерує сповіщення для персоналу.
      Нереалізовано: Було вирішено відмовитися від поділу на зони у кожному
музеї через велику складність реалізацію цієї ідеї.

      Висновки:

      Під час виконання проєкту ArtGuard було здійснено налаштування
серверного рішення через Ngrok, що дозволило інтегрувати IoT-пристрої та
здійснювати передачу даних у реальному часі до бази даних. Це спростило процес
тестування і демонстрації функціоналу, адже дало змогу працювати з локальним
сервером через інтернет. Також було проведено аналіз початкового плану проєкту і
порівняння з реально реалізованим функціоналом. Більшість завдань вдалося
успішно виконати, зокрема основні функції моніторингу,сповіщень, автоматичних
нагадувань та доступу до записів. Завдання, які не були реалізовані, були спрощені
або будуть реалізовані в майбутньому. Проєкт досягнув основних цілей на цю
мить, забезпечивши необхідну функціональність згідно з планом.
